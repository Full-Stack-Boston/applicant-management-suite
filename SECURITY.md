# Security Policy

## Supported versions

Security fixes are applied to the latest version on `main`. Older commits,
forks, and independently deployed instances are not supported.

## Reporting a vulnerability

Please do not open a public issue for a suspected vulnerability.

Use GitHub's private vulnerability reporting for this repository:

1. Open the repository's **Security** tab.
2. Select **Advisories**.
3. Select **Report a vulnerability**.

Include the affected component, reproduction steps, potential impact, and any
suggested mitigation. Maintainers will acknowledge a complete report as soon
as practical and coordinate disclosure after a fix is available.

## Deployment responsibility

This public repository contains a configurable application template. Operators
are responsible for securing their Firebase project, environment variables,
service accounts, third-party integrations, user permissions, and deployed
infrastructure. Never report real credentials or personal applicant data in an
issue, pull request, or vulnerability report.
